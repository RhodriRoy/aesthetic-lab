# UI/UX Pro Max —— 完整风格参考系统（AI 驱动构建成果）

## 1. 项目解决的核心痛点

在 UI/UX 设计工作中，设计师和开发者长期面临以下痛点：

- **风格选择困难**：面对 Neubrutalism、Glassmorphism、Cyberpunk 等数十种流行风格，缺乏系统化的对比与参考工具，难以快速确定项目视觉方向。
- **配色与字体割裂**：配色方案、字体配对、艺术效果分散在不同工具中，无法实时联动预览，导致"调完颜色发现字体不搭"的低效循环。
- **设计到开发的断层**：设计师定好风格后，开发者需要手动转换颜色值为 Tailwind/CSS/Figma Tokens 等格式，重复劳动且容易出错。
- **AI 提示词编写成本高**：使用 Midjourney、DALL-E、Stable Diffusion 等工具时，每次都需要从零编写包含风格、配色、字体、艺术效果的详细提示词，耗时费力。

**UI/UX Pro Max** 通过构建一站式风格灵感工作台，集成 20 种 UI 风格、22 种配色（含国风传统色）、中英文字体配对、18 种视觉艺术效果，并提供实时预览、一键 AI 提示词生成、多格式设计令牌导出，打通了"灵感探索 → 风格定义 → AI 辅助创作 → 开发落地"的完整链路。

## 2. 核心逻辑流与 AI 协作构建方式

本项目采用**多 Agent 协作 + 模块化长链推理**的方式完成构建，核心逻辑流如下：

### 阶段一：架构规划 Agent（长链推理）
- **输入**：项目需求文档（20 种风格、实时预览、提示词生成、导出功能）。
- **推理链**：单页应用架构 → 模块拆分（数据层 / 渲染层 / 状态管理层 / 导出层）→ URL 状态同步方案 → localStorage 持久化策略 → 移动端适配方案。
- **输出**：项目文件结构、模块接口定义、状态管理规范。

### 阶段二：数据建模 Agent（批量生成）
- **并行任务**：同时生成 20 种 UI 风格的完整数据对象（名称、氛围、关键词、prompt、字体搭配、walkthrough 文化溯源）。
- **质量校验**：通过交叉引用验证风格之间的关联性（如 Neo-Brutalism 与 Pop Art 的兼容性、Japanese Zen 与 Wabi-Sabi 的东方美学一致性）。
- **扩展生成**：国风配色（故宫红墙、敦煌飞天、三星堆青铜等 12 组）与艺术效果（波普、故障艺术、酸性设计等 18 种）的独立数据模块。

### 阶段三：前端开发 Agent（组件化实现）
- **配置注册表模式**：建立 `CONFIG_REGISTRY` 单点注册系统，统一管理风格 className、圆角、阴影、边框等视觉 token。
- **状态联动引擎**：实现风格 × 配色 × 字体 × 艺术效果的四维联动，任意维度切换时自动触发实时预览重渲染。
- **导出模块工厂**：基于当前状态动态生成 Tailwind config、CSS 变量、Figma Tokens（含 DTCG 格式）、SwiftUI / Compose 代码、多平台 AI 提示词。

### 阶段四：交互优化 Agent（体验打磨）
- **历史管理器**：实现 20 步撤销/重做栈，支持快捷键（Ctrl+Z / Ctrl+Shift+Z）。
- **智能联动推荐**：基于 `STYLE_LINKAGE` 映射表，当用户选择"三星堆青铜"配色时，自动推荐"三星堆·现代"艺术效果。
- **海报导出与分享**：通过 URL hash 编码完整状态，实现"一键复制链接，他人打开即见相同风格配置"的协作能力。

### 多 Agent 协作特征
- **无状态并行**：数据 Agent 与架构 Agent 独立运行，通过预定义的 JSON Schema 对接。
- **迭代反馈环**：预览 Agent 在检测到渲染异常时，自动回溯至配置注册表 Agent 修正 token 映射。
- **语义一致性校验**：所有 AI 生成的 prompt 与 walkthrough 文案，均经过风格一致性校验（如 Cyberpunk 必须包含 neon glow、terminal font 等关键词）。
